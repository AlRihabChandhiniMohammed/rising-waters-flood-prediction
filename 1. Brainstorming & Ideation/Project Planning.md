# Project Planning — Rising-Waters-Flood-Prediction

## Overview

The Rising Waters - Flood Prediction project aims to develop a machine learning-based web application that predicts the likelihood of floods using historical weather data. The project compares multiple classification algorithms, including Decision Tree, Random Forest, K-Nearest Neighbors (KNN), and XGBoost, and deploys the best-performing model using Flask. The application provides an easy-to-use interface for predicting flood risk based on weather parameters.

## Scope

### In Scope
- Collect and analyze historical flood and weather datasets.
- Perform data preprocessing and feature engineering.
- Train and compare Decision Tree, Random Forest, KNN, and XGBoost models.
- Select the best-performing model.
- Develop a Flask-based web application.
- Display flood prediction results through a user-friendly interface.

### Out of Scope
- Real-time weather data integration.
- Live satellite or IoT sensor integration.
- Mobile application development.
- Automatic emergency alert systems.

## Deliverables

- Entity Relationship Diagram (ER Diagram)

- Trained machine learning models

## Risks & Mitigation

| Risk | Mitigation Strategy |
|------|---------------------|
| Poor dataset quality | Perform data cleaning and preprocessing. |
| Model overfitting | Compare multiple models and validate using test data. |
| Integration issues | Test Flask application thoroughly before deployment. |
| Git merge conflicts | Use Git branches and frequent commits. |
| Deployment issues | Test the application locally before cloud deployment. |
